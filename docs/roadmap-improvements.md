# Roadmap: Качественные улучшения -- Diary of Feelings

> Версия 1.0, 2026-03-08
> Все фазы сборки (0–6) завершены. Этот роадмап -- про глубину, а не ширину.
> Принцип: **каждая фича должна работать отлично**, а не просто работать.

---

## Приоритеты

| Приоритет | Область | Почему |
|-----------|---------|--------|
| 🔴 Высокий | Задачи (Calendar + Sticky Notes) | Основная ежедневная точка входа |
| 🔴 Высокий | AI Ассистент | Ключевое отличие продукта |
| 🟡 Средний | Дневник эмоций | Эмоциональное ядро продукта |
| 🟡 Средний | Главная (Tree Graph) | Первый экран после логина |
| 🟢 Низкий | Заметки | Вспомогательный модуль |
| 🟢 Низкий | Отчёты | Полезен но не критичен |
| 🟢 Низкий | КПТ-дневник | Продвинутый модуль |

## Блок 1: Задачи -- Календарь и Стикеры 🔴

**Текущее состояние:** Базовый monthview + drag & drop (desktop + touch), DateTasksPanel.
**Цель:** Система планирования, которой хочется пользоваться каждый день.

### 1.1 Стикеры (визуал и взаимодействие)

- [ ] **Завершение задачи** -- тап по стикеру чекает его (зачёркнутый текст + fade)
- [ ] **Приоритет** -- маленький цветной флажок (красный/жёлтый/нет) в углу стикера
- [ ] **Счётчик задач на дате** -- бейдж `3` над ячейкой если задач > 1
- [ ] **Анимация создания** -- стикер "падает" на ячейку при drop (spring-эффект)
- [ ] **Long-press на стикере** -- контекстное меню: Изменить / Переместить / Удалить
- [ ] **Стикер в пуле** -- при нажатии открывается DateTasksPanel с выбором даты

### 1.2 Редактор задачи

- [ ] **Напоминание** -- datetime picker → Web Push уведомление
- [ ] **Цветовые категории** -- 5 цветов на выбор в редакторе (не только 3 пресета стикеров)
- [ ] **Краткое / развёрнутое** -- стикер показывает первые 2 строки, тап → полный текст
- [ ] **Время** -- опциональное поле "во сколько" (показывается на стикере как `14:30`)

### 1.3 Навигация по Calendar

- [ ] **Свайп по месяцам** -- touchstart/touchend для смены месяца (сейчас только кнопки)
- [ ] **"Сегодня" jump** -- кнопка в хедере возвращает к текущей дате
- [ ] **Неделя/месяц переключение** -- toggle Week view (7 ячеек) vs Month view

### 1.4 DateTasksPanel (список задач для даты)

- [ ] **Сортировка** -- drag & drop внутри панели для порядка задач
- [ ] **Checkbox прямо в панели** -- завершить без открытия редактора
- [ ] **Цвет задачи** -- полоска-акцент слева соответствующего цвета стикера
- [ ] **Пустой день** -- мотивирующий текст + иллюстрация вместо серой пустоты

## Блок 2: AI Ассистент 🔴

**Текущее состояние:** Базовый чат, Ollama → Claude → mock fallback.
**Цель:** Ощущение настоящего персонального помощника, который тебя знает.

### 2.1 UX чата

- [ ] **Стриминг ответа** -- текст появляется постепенно (SSE или pseudo-stream с задержкой)
- [ ] **Typing indicator** -- три точки пока ассистент "думает"
- [ ] **Conversation starters** -- чипы-подсказки при пустом чате: "Как я себя чувствую на этой неделе?", "Помоги составить план", "Разбор тревожной ситуации"
- [ ] **Quick replies** -- ассистент может предложить кнопки-ответы (Yes/No/Подробнее)
- [ ] **Scroll to bottom** -- кнопка ↓ при прокрутке вверх по истории
- [ ] **Copy bubble** -- long-press на сообщении → Скопировать

### 2.2 Контекст и персонализация

- [ ] **Context card** -- маленький коллапсируемый блок вверху: что ассистент знает о тебе сейчас (эмоция дня, задачи, заметки)
- [ ] **Дата последней сессии** -- разделитель между днями в истории чата (`── Вчера ──`)
- [ ] **Имя пользователя** -- ассистент обращается по имени из профиля
- [ ] **Моод-опрос при открытии** -- если эмоция не записана сегодня, мягкий вопрос первым сообщением

### 2.3 Качество ответов (backend)

- [ ] **Системный промпт v2** -- добавить в контекст: имя, текущие задачи, последние 3 эмоции, КПТ-карточки в работе
- [ ] **Ограничение истории** -- передавать последние 20 сообщений (не всю историю)
- [ ] **Структура ответа** -- промпт требует: короткий абзац + конкретное действие
- [ ] **Язык** -- промпт фиксирует русский язык ответов

## Блок 3: Дневник эмоций 🟡

**Текущее состояние:** Колесо эмоций, запись (intensity + note), история, статистика.
**Цель:** Ежедневный ритуал с ощущением прогресса и смысла.

### 3.1 Запись эмоции

- [ ] **Одна запись в день (мягко)** -- если запись уже есть, показать: "Ты уже отметил(а) своё состояние. Хочешь обновить?"
- [ ] **Анимация после записи** -- пульсация + краткое сообщение ("Записано. Держись!") перед закрытием
- [ ] **Тайм-капсула UX** -- checkbox "Показать через 30 дней" в форме, визуальный замок на карточке
- [ ] **Голосовая заметка** -- кнопка 🎤 для записи аудио-заметки к эмоции (MediaRecorder API)

### 3.2 История

- [ ] **Timeline view** -- вертикальная лента вместо просто списка (дата слева, карточка справа)
- [ ] **Цветовое кодирование** -- левая полоска цвета эмоции (радость → жёлтый, тревога → синий и т.д.)
- [ ] **Раскрыть/свернуть** -- карточка по умолчанию свёрнута, тап → разворачивает заметку
- [ ] **Быстрый фильтр** -- чипы: Всё / Эта неделя / Этот месяц / По эмоции

### 3.3 Паттерны и статистика

- [ ] **Streak счётчик** -- "7 дней подряд 🔥" на главной и в хедере дневника
- [ ] **Недельный мини-граф** -- 7 цветных точек (дни недели) в заголовке страницы
- [ ] **Открытие тайм-капсулы** -- если прошло 30 дней, уведомление + анимированное "вскрытие" (конверт)
- [ ] **Топ эмоций недели** -- три самые частые эмоции как цветные пилюли

## Блок 4: Главная (Tree Graph) 🟡

**Текущее состояние:** Приветствие + базовый SVG tree, animated orbs.
**Цель:** Живой дашборд, который даёт смысл заходить на главную.

### 4.1 Персонализация приветствия

- [ ] **Время суток** -- "Доброе утро, Миша" / "Добрый вечер" в зависимости от часа
- [ ] **Мотивационная фраза** -- меняется каждый день (массив из 30+ фраз с посевом по дате)
- [ ] **Эмоция дня** -- если записана: emoji + название рядом с приветствием
- [ ] **Streak badge** -- число дней подряд с записями

### 4.2 Tree Graph

- [ ] **Счётчики на ветках** -- "Задачи (3)" / "Эмоции (7)" -- число активных элементов
- [ ] **Нода с задачами** -- показывает 1–2 ближайшие задачи как маленькие чипы на ветке
- [ ] **Нода с эмоциями** -- показывает эмоцию дня (цвет ноды = цвет эмоции)
- [ ] **Анимация idle** -- ветки тихо покачиваются (GSAP sin-цикл, амплитуда 2px)
- [ ] **Tap feedback** -- при тапе на ветку: пружинный отскок перед переходом

### 4.3 Быстрые действия

- [ ] **FAB (floating action button)** -- раскрывается в 2 кнопки: "Добавить задачу" / "Записать эмоцию"
- [ ] **Quick emotion** -- мини-колесо прямо с главной (без перехода в раздел Эмоции)

## Блок 5: Заметки 🟢

**Текущее состояние:** Список карточек, inline-редактор, поиск.
**Цель:** Удобный блокнот, который приятно открывать.

### 5.1 Редактор

- [ ] **Markdown preview** -- `**жирный**`, `*курсив*`, `# заголовок` рендерятся как rich text
- [ ] **Автосохранение** -- debounce 1.5s, индикатор "Сохранено"
- [ ] **Пустые строки** -- новая заметка открывается с placeholder "О чём думаешь?"

### 5.2 Список

- [ ] **Закреплённые заметки** -- pin → всегда первые в списке
- [ ] **Превью** -- первые 2–3 строки текста в карточке (сейчас только заголовок)
- [ ] **Дата** -- относительная ("Вчера", "3 дня назад") в карточке
- [ ] **Цветные заметки** -- как Google Keep: выбор фона карточки из 6 цветов

### 5.3 Поиск

- [ ] **Highlight matches** -- найденные слова подсвечиваются в превью
- [ ] **Поиск по тегам** -- `#тег` как фильтр, автодополнение

## Блок 6: Отчёты 🟢

**Текущее состояние:** SVG line chart, period selector, backend stats.
**Цель:** Отчёт, который реально читают и понимают.

### 6.1 Визуализация

- [ ] **Heatmap эмоций** -- calendar heatmap (как GitHub contribution graph) за последние 3 месяца
- [ ] **Bar chart эмоций** -- топ эмоций за период как горизонтальные бары с цветами
- [ ] **Trend line** -- скользящее среднее на линейном графике (сглаживание)
- [ ] **Пустое состояние** -- если данных мало: иллюстрация + "Веди дневник несколько дней чтобы увидеть паттерны"

### 6.2 Insights

- [ ] **AI-сводка периода** -- 2–3 предложения от ассистента: что заметно в данных
- [ ] **Лучший день недели** -- "В среду ты обычно чувствуешь себя лучше всего"
- [ ] **Корреляция задачи↔эмоции** -- "В дни с 3+ завершёнными задачами настроение выше"

### 6.3 UX

- [ ] **Шаринг** -- кнопка "Поделиться" генерирует картинку отчёта (html2canvas)
- [ ] **Период произвольный** -- date range picker (сейчас только неделя/месяц/3 месяца)

## Блок 7: КПТ-дневник 🟢

**Текущее состояние:** Kanban 3 колонки, drag & drop, базовый редактор СМЭР.
**Цель:** Настоящий CBT-инструмент для регулярной работы с мышлением.

### 7.1 Карточка СМЭР

- [ ] **Краткий / полный режим** -- в колонке видно только Ситуацию, клик → детали
- [ ] **Прогресс-бар** -- визуально показывает заполненность (из 4 базовых полей)
- [ ] **Дата создания** -- "3 дня назад" на карточке
- [ ] **Тег когнитивного искажения** -- цветной пилюль на карточке если выбрано

### 7.2 Расширенный режим

- [ ] **Toggle "Продвинутый режим"** -- показывает поля: Искажение / Оспаривание / Альтернативная мысль
- [ ] **Picker искажений** -- визуальная сетка с иконками (не просто dropdown)
- [ ] **Интенсивность до/после** -- два слайдера: эмоция при создании vs после проработки
- [ ] **Рефлексия** -- текстовое поле "Что понял(а)?" только в колонке "Проработана"

### 7.3 Канбан

- [ ] **Счётчик в заголовке колонки** -- "В работе (2)"
- [ ] **Архив** -- кнопка "Показать проработанные" скрывает/открывает завершённые карточки
- [ ] **Напоминание** -- если карточка в "В работе" больше 3 дней, лёгкая подсветка

## Блок 8: Технические улучшения (Cross-cutting) 🟢

### 8.1 Тёмная тема

- [ ] **Аудит всех экранов** -- пройти каждый экран в dark mode, найти белые/жёсткие элементы
- [ ] **Тёплые тёмные цвета** -- фон не `#111` а `#1a1520` (тёплый тёмно-фиолетовый)
- [ ] **Glassmorphism в dark** -- opacity и blur подобрать отдельно для тёмной темы

### 8.2 Производительность

- [ ] **Виртуальный список** -- история эмоций и заметки при 100+ элементах (vue-virtual-scroller)
- [ ] **Lazy images** -- `loading="lazy"` и intersection observer для тяжёлых блоков
- [ ] **Route prefetch** -- prefetch следующего likely-роута при hover на bottom nav кнопке

### 8.3 Мобильная полировка

- [ ] **Safe area insets** -- `env(safe-area-inset-bottom)` везде где нужен padding под bottom nav
- [ ] **Haptic feedback** -- `navigator.vibrate(10)` при drag drop и других ключевых действиях
- [ ] **Pull-to-refresh** -- свайп вниз в списках обновляет данные
- [ ] **Keyboard avoidance** -- форма не перекрывается клавиатурой (Visual Viewport API)

### 8.4 Доступность (a11y)

- [ ] **Focus visible** -- все интерактивные элементы видны при навигации с клавиатуры
- [ ] **ARIA labels** -- иконочные кнопки без текста имеют `aria-label`
- [ ] **Contrast audit** -- текст на glassmorphism-фонах проходит AA (4.5:1)
- [ ] **Reduced motion** -- уважать `prefers-reduced-motion` для GSAP анимаций
