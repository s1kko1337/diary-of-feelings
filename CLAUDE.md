# Diary of Feelings — Мотивационный Ежедневный Ассистент

## Суть проекта

Личное пространство для ежедневной рефлексии и эмоциональной поддержки. Не трекер ради цифр — инструмент для понимания себя. Пастельный, спокойный дизайн. Никакого давления на пользователя.

### Ключевые модули

- **Ежедневные статьи** — 3 статьи в день (разогрев → глубина → рефлексия)
- **Дневник эмоций** — колесо эмоций, одна строка в день, тайм-капсулы, паттерны
- **КПТ-дневник (СМЭР)** — структурированный дневник по модели когнитивно-поведенческой терапии
- **AI-ассистент** — советы на основе прочитанного, эмоций и истории (тёплый тон, без пафоса)
- **Трекер задач** — контекстный планировщик с намерениями
- **Трекер времени** — лёгкий, для осознанности
- **Заметки и идеи** — быстрый блокнот с тегами
- **Личная библиотека** — избранные статьи, советы, записи
- **Стрики** — мягкие, без давления
- **Уведомления** — одно в день, как письмо

### Дневник эмоций (подробнее)

Лёгкий ежедневный чекин эмоционального состояния:

- **Колесо эмоций** — пользователь выбирает точную эмоцию (тревога, усталость, лёгкость, благодарность и т.д.), а не просто «хорошо/плохо»
- **Одна строка в день** — минимальный барьер входа, одно предложение уже запись
- **Интенсивность** — шкала 0–10
- **Тайм-капсула** — запись, которую увидишь через 30 дней
- **Паттерны** — раз в неделю AI замечает закономерности: «Ты три раза упоминал тревогу по понедельникам»
- **Месячный отчёт** — как менялось настроение, какие темы повторялись

### КПТ-дневник — СМЭР (подробнее)

Структурированный дневник на основе когнитивно-поведенческой терапии (модель ABC / СМЭР). Помогает осознавать связь между ситуациями, мыслями и эмоциями, выявлять когнитивные искажения и формировать более адаптивное мышление.

**Структура записи СМЭР:**

| Поле | Описание | Пример |
|------|----------|--------|
| **С — Ситуация** | Что произошло (объективно, без оценок) | «Коллега не ответил на моё сообщение за 3 часа» |
| **М — Мысли** | Автоматические мысли, которые возникли | «Он меня игнорирует», «Я сделал что-то не так» |
| **Э — Эмоции** | Какие эмоции и их интенсивность (0–10) | Тревога 7/10, обида 5/10 |
| **Р — Реакция** | Что сделал / как себя повёл | «Написал ещё 3 сообщения подряд» |

**Дополнительные поля (продвинутый режим):**

- **Когнитивное искажение** — выбор из списка: катастрофизация, чтение мыслей, чёрно-белое мышление, персонализация, сверхобобщение и др.
- **Оспаривание** — «Есть ли доказательства этой мысли?», «Как бы я посоветовал другу?»
- **Альтернативная мысль** — более реалистичная интерпретация: «Возможно, он занят на встрече»
- **Эмоции после** — повторная оценка интенсивности после проработки

**Принципы UX:**

- Базовый режим (С + М + Э) для новичков, продвинутый (полный СМЭР + оспаривание) для опытных
- Никакого давления — можно заполнить только ситуацию и эмоцию
- AI может подсказать когнитивное искажение и помочь с оспариванием
- Записи используются AI-ассистентом для более точных советов

## Архитектура

### API слой

- `src/api/*.js` — обёртки, вызывают mock через `request(handler, { delay, timeout })`
- `src/api/mock/*.js` — данные и генераторы, хранение в localStorage
- `src/api/client.js` — `ApiError`, `request()` с эмуляцией задержки и таймаутом

### Stores (Pinia composition)

- Только composition syntax: `defineStore('name', () => { ... })`
- Store = глобальное состояние; локальное — в компоненте/composable
- Каждый модуль — свой store в `src/stores/`

### Composables

- Файл и функция: `useXxx.js` → `export function useXxx()`
- Возвращают ref/computed, без UI-логики
- `src/composables/`

### Темы

- CSS-переменные в `src/assets/theme.css` (light по умолчанию, `.dark` класс)
- Шрифты: Inter (основной), JetBrains Mono (метрики)
- Composable `useChartColors()` — цвета графиков из CSS-переменных

### Структура проекта


## Code Conventions

### Vue Components

- Всегда `<script setup>` — никогда Options API
- Порядок блоков: `<template>` → `<script setup>` → `<style scoped>`
- Стили всегда `scoped`, глобальные — только в `assets/main.css`
- Компоненты — PascalCase файлы, props — camelCase в JS / kebab-case в шаблонах
- Emit-события — kebab-case (`@update-value`)
- **НЕ использовать `;` в `@click`** — выносить в метод

### Pinia Stores

- Только composition syntax: `defineStore('name', () => { ... })`
- Store = глобальное состояние; локальное — в компоненте/composable

### Routing

- Lazy-load для всех страниц: `component: () => import('@/views/XxxView.vue')`
- Имена маршрутов — kebab-case

### Imports

- Алиас `@` = `src/`
- Vue/библиотеки — первыми, потом внутренние модули

### Styling

- Tailwind CSS 4 — utility-first, `@apply` только в крайнем случае
- Не использовать inline-стили
- Цвета, шрифты, spacing — через Tailwind-токены и CSS-переменные

### Дизайн-система

**Философия:** Лёгкий, прогрессивный, не-CRM дизайн. Вдохновение — wellness-приложения (Calm, Headspace), editorial-стили (non-linear.studio), тренды 2026 (glassmorphism, 3D-орбы, liquid design).

**Навигация:**
- Десктоп: плавающий pill-навбар сверху по центру (`AppNavbar.vue`), glassmorphism
- Мобилка: минимальный хедер (`AppHeader.vue`) + floating bottom nav (`AppBottomNav.vue`)
- Никаких сайдбаров — контент по центру, `max-width: 720px`

**Glassmorphism (ключевые правила):**
- `backdrop-filter: blur(20–24px)` + полупрозрачный фон (`rgb(255 255 255 / 0.7)`)
- Тонкие бордеры (`1px solid` с opacity 10–15%)
- Мягкие цветные тени (лавандовые, не чёрные)
- Не больше 5 стекло-слоёв на экране (GPU-нагрузка)
- НЕ анимировать backdrop-filter — дорого для рендеринга
- Всегда добавлять `-webkit-backdrop-filter` для Safari

**3D и декоративные элементы:**
- Фоновые орбы: `radial-gradient` + `blur(60–70px)` + `position: fixed` + анимация `float-slow`
- Hero-орб: gradient (лаванда → персик → мята) + glow через box-shadow + пульсация
- Карточки: мягкие gradient-орбы в углах (декоративные, opacity 0.5)
- Все декоративные элементы — `pointer-events: none`

**Палитра:**
- Основные: лаванда (#c4b5e0), персик (#f8c9a0), мятный (#a8dfc8), розовый (#f0b8c0)
- Каждый цвет имеет `-soft` вариант (opacity 10–20%) для фонов
- Тёмная тема — тёплая, глубокая (не холодная), те же акценты с адаптированной яркостью
- Фон — градиент, не однотон: `linear-gradient(145deg, cream → lavender-tint → mint-tint)`

**Типографика:**
- Inter — основной, system-ui fallback
- Rounded geometric sans-serif — гуманный, не клинический
- Заголовки: крупные (2–2.5rem), weight 700, letter-spacing -0.03em
- Labels: uppercase, 0.8rem, letter-spacing 0.06em, muted color
- Тело: 0.9rem, line-height 1.65

**Анимации и микровзаимодействия:**
- Плавные transition (0.2–0.3s ease) на hover
- Hover на карточках: `translateY(-2px)` + усиление тени
- Ссылки-действия: `gap` увеличивается при hover (стрелка «едет»)
- Фоновые орбы: медленная анимация drift (20–25s cycle)
- Hero-орб: мягкая пульсация scale (4s cycle)
- Никаких резких, aggressive анимаций — всё спокойное

**Компоненты (паттерны):**
- Карточки: `.glass-card` / `.glass-card-elevated` (глобальные классы в `main.css`)
- Pill-кнопки: `border-radius: var(--radius-full)`, цветной фон + бордер
- Иконки: lucide-vue-next, `stroke-width: 1.8`, size 18–22
- Все radius через CSS-переменные (`--radius-sm` .. `--radius-full`)

### Formatting & Linting

- Prettier: без точки с запятой, одинарные кавычки, 100 символов
- Перед коммитом: `npm run lint && npm run format`

### Язык

- Код, комментарии, коммиты: **английский**
- UI-текст, документация: **русский**, без англицизмов (в коде `alerts`, в UI — «уведомления»)

### AI-интеграция

- Тон AI: тёплый, между хорошим другом и дзен-монахом
- Ответ AI: короткий абзац + одно конкретное действие
- Контекст AI: прочитанное сегодня + эмоция + история 7 дней
- При разработке использовать моки вместо реальных API-вызовов
